A Tampermonkey userscript for collecting, exporting, and downloading accessible iCourse playback video links from a logged-in browser session at `https://icourse.fudan.edu.cn/`.

## Features

- Loads the course list visible to the current account and groups courses by term.
- Supports selecting one session, all sessions in one course, or all replayable sessions across courses.
- Supports multiple video streams, with the main stream selected by default and an option to select all streams.
- Opens playback pages and collects signed MP4 URLs generated by the player.
- Exports JSON, CSV, TXT, PowerShell, parallel PowerShell, aria2c input, and IDM PowerShell files.
- Sends tasks to Motrix/aria2 through local aria2 JSON-RPC.
- Optionally auto-confirms the platform resource usage notice on playback pages.

## Installation

1. Install a userscript manager:
   - [Tampermonkey](https://www.tampermonkey.net/)
   - [Violentmonkey](https://violentmonkey.github.io/)
2. Install the userscript using either source:
   - GitHub Raw: `https://raw.githubusercontent.com/TimoZhou1024/icourse_download/main/icourse-batch-downloader.user.js`
   - Greasy Fork: `https://greasyfork.org/zh-CN/scripts/583017-icourse-batch-video-downloader/`
3. Open `https://icourse.fudan.edu.cn/` and log in normally.
4. Use the `iCourse 视频` button in the bottom-right corner of the page.

## Usage

1. Log in to iCourse and open a page where your courses are accessible.
2. Click the `iCourse 视频` button in the bottom-right corner.
3. Confirm the compliance notice on first use: only process resources you are authorized to access and allowed to save.
4. Select courses, sessions, or individual video streams.
5. Click `收集所选链接` and wait for the script to capture signed URLs from playback pages.
6. Choose a download/export method:
   - `浏览器下载`
   - `发送到 Motrix/aria2`
   - `导出下载 PowerShell`
   - `导出并行 PowerShell`
   - `导出 aria2c 清单`
   - `导出 IDM PowerShell`

## Download Methods

### Browser Download

Suitable for a small number of MP4 files. HLS/m3u8 streams are not merged in the browser; use external tools for those.

### Motrix / aria2

The script can send tasks to Motrix/aria2. The default RPC endpoint is:

```text
http://127.0.0.1:16800/jsonrpc
```

The common RPC endpoint for standalone aria2 is:

```text
http://127.0.0.1:6800/jsonrpc
```

iCourse signed MP4 URLs are unstable with external multi-connection Range requests. The script uses one connection per file and improves throughput by downloading multiple files concurrently:

```text
split=1
max-connection-per-server=1
continue=false
async-dns=false
no-proxy=*
```

If Motrix cannot receive tasks, make sure aria2 RPC is enabled and check the RPC URL/token.

### IDM

Use `导出 IDM PowerShell` to generate a script. The script locates `IDMan.exe`, adds signed MP4 URLs to the IDM queue, and starts downloading.

IDM command-line mode does not attach per-task `Referer` or `User-Agent` headers, so collect signed MP4 URLs before exporting.

### PowerShell

- `导出下载 PowerShell`: downloads with `curl.exe`; slower but relatively stable.
- `导出并行 PowerShell`: requires PowerShell 7 for true parallelism. Tune `$ThrottleLimit` in the exported script.

## Known Limitations

- The script does not bypass login, permissions, CAPTCHA, DRM, or platform restrictions.
- Raw MP4 URLs usually cannot be downloaded directly; signed URLs with `clientUUID` and `t` parameters must be captured first.
- Signed URLs may expire. Use `刷新 signed URL` to capture fresh URLs.
- HLS/m3u8 streams are not merged in the browser.
- iCourse signed MP4 responses may advertise `Accept-Ranges: bytes`, but external Range requests can return `403`; multi-connection segmented downloading for a single file is not recommended.
